[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's walk through the architecture. 
Every user request starts at the START node and immediately hits the security_checkpoint node. 
If the text contains sensitive PII like Social Security Numbers, or prompt injection attempts, it's flagged and routed straight to the final output node. 
Clean tickets pass to the orchestrator agent. 
The orchestrator agent is our primary manager; it delegates the classification to the ticket_classifier sub-agent to determine priority and category.
It also interacts with our MCP Server tools to gather system status, check billing info, or search our troubleshooting knowledgebase.
Then, the orchestrator delegates the actual writing of the reply to the resolution_drafter sub-agent.
Once the draft is ready, the review_decision node determines if it requires human verification. 
If priority is High, it routes to the human_review node where the workflow pauses, prompting a human agent to review and approve or reject the draft resolution.
Otherwise, it is automatically approved and returned to the user via the final_output node.
